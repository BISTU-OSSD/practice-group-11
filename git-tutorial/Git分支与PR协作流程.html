Git分支与PR协作流程 一、分支使用规范

  1. main为主稳定分支，仅存放审核完成的正式内容，禁止直接在此分支修改提交
  2. 个人开发分支统一采用feat/功能名格式命名，本次分支为feat/docs-structure
  3. 每人仅操作自身分工文件，不随意改动他人配置与文档 二、日常开发步骤
  4. 先同步主分支最新代码：git pull origin main
  5. 切换至自己的开发分支进行编辑
  6. 内容修改完毕后依次执行： git add . git commit -m "说明本次修改内容" git push origin 对应分支名
  7. 全部完成后在GitHub仓库发起PR合并请求，等待组长审核通过后并入主分支 三、PR提交注意事项
  8. 提交备注清晰写明新增、修改的文档范围
  9. 仅上传分工内文件，不改动无关代码
  10. 审核出现问题，在当前分支修正后重新推送更新PR
